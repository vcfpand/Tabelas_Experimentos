# Contribuindo

Obrigado pelo interesse em contribuir com a **Planilha de Experimento**!

Este projeto é mantido como parte de uma pesquisa acadêmica, mas contribuições externas são bem-vindas. Antes de começar, leia este guia.

## 🐛 Reportando Problemas

Encontrou um bug ou tem uma sugestão? Abra uma [issue](https://github.com/vcfpand/Tabelas_Experimentos/issues) com:
- Descrição clara do problema ou sugestão.
- Passos para reproduzir (se aplicável).
- Navegador e sistema operacional.
- Capturas de tela, se possível.

## 💡 Sugestões de Melhorias

Ideias para novas funcionalidades são muito bem-vindas. Utilize a tag `enhancement` nas issues.

## 🔧 Enviando Código

1. Faça um fork do repositório.
2. Crie um branch para sua feature: `git checkout -b minha-feature`
3. Faça commit das alterações: `git commit -m 'Adiciona nova feature'`
4. Envie para o branch: `git push origin minha-feature`
5. Abra um Pull Request.

### Estilo de Código
- **JavaScript:** ES6+ com `import`/`export`. Use `const` e `let`.
- **CSS:** Organize por seções comentadas. Mantenha variáveis CSS.
- **HTML:** Estrutura semântica e acessível.
- **Python:** PEP 8. Use type hints quando possível.

### Antes de Enviar
- Teste localmente com `index.html` (aplicativo web) e `streamlit run app.py` (dashboard).
- Certifique-se de que não há erros no console do navegador.
